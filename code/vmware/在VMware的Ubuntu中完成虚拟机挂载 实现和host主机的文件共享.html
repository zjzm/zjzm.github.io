VMware中Linux虚拟机挂载主机共享文件夹的方法

On this page

在VMware中设置好共享文件夹之后，在Ubuntu中是看不到这个文件的。 共享文件夹的其实在目录/mnt/hgfs下 这时先看一下虚拟机能不能找到这个文件，终端直接输入vmware-hgfsclient会返回你设置的文件夹。 此时进入root模式！！！ 如果之前设置过密码，可以直接su -root进入 如果没有设置过密码，这里的密码不是开机的用户密码，所以你发现即使密码没错还是不对。先设置密码sudo passwd root root模式和普通模式的对比：

在root模式下sudo mount -t fuse.vmhgfs-fuse .host:/ /mnt/hgfs -o allow_other，没有必要把host:/后面改成自己具体的文件夹。 这样每次重启都要输入一遍，如果不想重新输入需要在root模式下 输入vi /etc/fstab，然后在最后一行加入.host:/ /mnt/hgfs fuse.vmhgfs-fuse allow_other 0，并保存。

但是这样有个问题就是重启之后可能发现自己在可视化界面里不能直接点击进去了，会提示权限不足不能访问。这是因为我们之前的操作是在root下的，而点击操作的是当前用户，此时我们需要把当前用户设置为root用户或者提升对这个文件的权限。 终端直接输入sudo visudo,这里不用vim打开或者gedit打开是因为vi更能够保护sudo的安全。 找到root权限，下面增加自己的用户名。注意用户名和ALL之间是tab不是空格。

最后reboot重启之后发现共享文件已经在里面并且可以直接打开了。

tips：可能是旧版本的问题，以前可能出现VMwaretools出现问题，看到需要sudo apt-get install open-vm-dkms，但是我这样只会显示无法定位文件夹，因为根本不需要这样！ 如果出现Error: cannot mount filesystem: No such device，更大的可能性是你:/host后面加了文件名，不要加!
